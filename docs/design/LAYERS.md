## 每层职责（一句话）

| 层号 | 名称 | 回答的问题 | 对上层提供什么 |
|------|------|-----------|---------------|
| **0** | 基础层 | 坐标怎么算？ | WGS84常量、时间网格、坐标变换 |
| **1** | 轨道层 | 卫星在哪？ | `(N×T×3)` 位置矩阵 |
| **2** | 链路层 | 能不能连上？ | ISL/GSL 可行性、距离、仰角 |
| **3** | 拓扑层 | 谁和谁连接？ | 邻接矩阵、ISL图 |
| **4** | 路由层 | 数据走哪条路？ | 最短路径、延迟 |
| **5** | 流量层 | 网络负载多重？ | 链路利用率、拥塞状态、电池 SOC |
| **6** | 优化层 | 怎么改进？ | 梯度方向、瓶颈检测、损失值 |
| **7** | 资源层 | 卫星有多少资源？ | 缓存/CDN/计算/传感器状态 |
| **8** | 安全层 | 攻击者怎么破坏？ | 恶意流量注入、故障注入 |
| **9** | 协议栈层 | TCP/BP 行为？ | 包级延迟、丢包、拥塞窗口 |
| **10** | 部署层 | 怎么变成真网络？ | Lima VM 配置、OEF 通道命令 |
| **11** | 实验层 | 实验怎么做？ | 配置→执行→保存→导出 |
| **12** | 交互层 | 用户怎么用？ | CLI / Agent / DSL |

## 关键设计规则

1. **下层永远不调用上层** — Layer 1 不知道 Layer 3 的存在
2. **同层可有多实现** — Layer 1 有 3 套传播器（TwoBody/SGP4/Keplerian），通过 Julia 多分派切换
3. **Layer 6/7/8 是并行层** — 它们都消费 Layer 0-5，但互相不依赖：
   - 优化层分析"怎么改进？"
   - 资源层描述"卫星有什么？"
   - 安全层模拟"攻击者怎么做？"
4. **Layer 9/10 是横向层** — 消费 Layer 0-4，与主流水线并行，按需加载
5. **Layer 11/12 在最上层** — 编排所有下层，用户只接触这两层

## 一个安全实验的调用链

```
Layer 12: @study "energy_drain" begin ... end     ← 声明
Layer 11: run_experiment(cfg)                      ← 执行
  ├── Layer 1: propagate_positions()               ← 卫星在哪
  ├── Layer 2: evaluate_isl_batch()                ← 能不能连
  ├── Layer 3: generate_topology(GridPlus())       ← 谁和谁连
  ├── Layer 4: route(Dijkstra(), ...)              ← 走哪条路
  ├── Layer 5: evaluate_traffic()                  ← 负载多重
  ├── Layer 7: evolve_power_state()                ← 电池消耗
  ├── Layer 8: inject_energy_drain()               ← 注入攻击
  └── Layer 6: bottleneck_snapshots()              ← 检测瓶颈
Layer 11: to_markdown(result)                      ← 导出
```
